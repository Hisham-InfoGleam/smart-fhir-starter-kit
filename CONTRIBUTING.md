# Contributing to SMART FHIR Starter Kit

Thank you for considering contributing to this project! Contributions help make this toolkit better for the healthcare developer community.

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs or suggest features
- Include clear steps to reproduce bugs
- Provide context about your environment (OS, Node version, browser)

### Submitting Changes

1. **Fork** the repository
2. **Create a branch** for your feature or fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes** with clear, descriptive commits
4. **Test** your changes locally
5. **Submit a Pull Request** with a clear description

### Code Guidelines

- Keep code simple and readable
- Add comments for complex logic
- Update documentation if you change behavior
- Use synthetic/sample data only (no real PHI)

### Documentation

- Use clear, community-focused language
- Avoid jargon where possible
- Include code examples when helpful

## Code of Conduct

- Be respectful and constructive
- Welcome newcomers and help them learn
- Focus on the code, not the person

## Questions?

Open a GitHub Issue or start a Discussion. We're happy to help!
